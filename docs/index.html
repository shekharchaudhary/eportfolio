Overview

This GitHub Pages site is my CS‑499 capstone ePortfolio. It presents a cohesive plan, the three capstone enhancements (Software Design & Engineering, Algorithms & Data Structures, and Databases), outcome mapping, and links to artifacts and evidence.

Design & Engineering

Refactor for modularity, reliability, and maintainability. Apply SOLID, add structured logging, better error handling, and test coverage.

Algorithms & Data

Identify hot paths and replace naive routines with efficient structures and algorithms. Provide complexity analysis and benchmarks.

Databases

Normalize schema, optimize queries with indexes, use transactions, and strengthen security via parameterization and least privilege.

Plan & Milestones

  1. Module 2 — Milestone 1: Code review of candidate artifacts, gap analysis, and enhancement plan with acceptance criteria and risks.
  2. Module 3 — Milestone 2: Enhancement 1 (Design & Engineering): refactor architecture, improve error handling, logging, configuration, and tests.
  3. Module 4 — Milestone 3: Enhancement 2 (Algorithms & Data Structures): optimize core routines; analyze complexity; benchmark improvements.
  4. Module 5 — Milestone 4: Enhancement 3 (Databases): schema updates, indexes, transactions, and security hardening.
  5. Module 7 — Final: Publish ePortfolio with artifacts, narratives, diagrams, and Professional Self‑Assessment.

Course Outcomes

CO1 — Collaboration

Use issues/PRs, decision records, and audience‑tailored communication.

CO2 — Communication

Clear README, architecture diagrams, ADRs, and API docs for mixed audiences.

CO3 — Algorithmic Design

Show choices, Big‑O analysis, benchmarks, and trade‑offs with evidence.

Apply disciplined workflows: lint, tests, CI, and automation to deliver value.

Threat model, validate inputs, protect data, least privilege, and safe errors.

Development Focus

Testing Strategy

Unit tests for core logic and edge cases, integration tests across modules, system tests for workflows, and benchmarks for algorithmic changes.

Security

Identify threats and mitigations; parameterize queries/ORM, handle secrets, enforce least privilege, and sanitize error handling.
